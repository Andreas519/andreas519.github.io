DAC – Digital-Analog-Wandler

Ein DAC wandelt einen digitalen Zahlenwert in eine analoge Ausgangsspannung um. DAC bedeutet Digital-Analog-Converter.

Grundprinzip

Der Mikrocontroller gibt intern eine Zahl vor. Der DAC erzeugt daraus eine passende Spannung am Ausgang. 

0     → ca. 0 V
128   → ca. halbe Betriebsspannung
255   → ca. maximale Ausgangsspannung

Beim klassischen ESP32 arbeitet der DAC mit 8 Bit. Dadurch sind 256 Spannungsstufen möglich.

DAC-Pins beim klassischen ESP32

  • GPIO25 → DAC1
  • GPIO26 → DAC2

Achtung: Nicht alle ESP32-Varianten besitzen einen DAC. Viele neuere Typen wie ESP32-C3, ESP32-C6 oder ESP32-S3 haben keinen klassischen DAC-Ausgang.

Beim klassischen ESP32 liegen die DAC-Ausgänge auf GPIO25 und GPIO26. Der DAC funktioniert grundsätzlich, kann aber bei erneutem Programmstart in Thonny den Fehler OSError: (-259, 'ESP_ERR_INVALID_STATE') erzeugen. In der verwendeten MicroPython-Firmware besitzt das DAC-Objekt keine Methode deinit(). Deshalb hilft bei diesem Fehler meist ein Reset des ESP32.

MicroPython-Beispiel

from machine import Pin, DAC
import time

dac = DAC(Pin(25))   # DAC1 beim klassischen ESP32

while True:
    dac.write(0)     # ca. 0 V
    time.sleep(1)

    dac.write(128)   # ca. halbe Spannung
    time.sleep(1)

    dac.write(255)   # ca. maximale Spannung
    time.sleep(1)

Wichtiger Hinweis

Der DAC-Ausgang kann nur wenig Strom liefern. Er eignet sich gut für hochohmige Eingänge oder Messzwecke, aber nicht zum direkten Betreiben größerer Lasten wie Lautsprechern, Motoren oder hellen LEDs.

Unterschied zu PWM und ADC

Begriff Richtung Aufgabe
ADC analog → digital misst eine Spannung
DAC digital → analog erzeugt eine Spannung
PWM digital → scheinbar analog schaltet sehr schnell ein und aus

Merksatz

ADC misst Spannung.
DAC erzeugt Spannung.
PWM täuscht eine analoge Spannung durch schnelles Schalten vor.
