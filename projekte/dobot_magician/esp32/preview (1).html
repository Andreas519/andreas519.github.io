2. Aufgabe der ESP32-CAM

Die ESP32-CAM übernimmt nur vier grundlegende Aufgaben:

  1. Kamera initialisieren,
  2. WLAN-Verbindung herstellen,
  3. Bild aufnehmen,
  4. Bild oder Videostream per HTTP bereitstellen.

Variante A: Einzelbilder

Python fordert Bild an
        ↓
ESP32-CAM nimmt aktuelles Bild auf
        ↓
JPEG wird übertragen
        ↓
Python verarbeitet genau dieses Bild
        ↓
Dobot führt Aktion aus

Typischer Endpunkt:

http://192.168.1.123/capture

Für die Robotersteuerung ist das besonders sinnvoll, weil der Dobot während einer Aufnahme möglichst stillstehen sollte.

Dobot steht still
      ↓
Bild aufnehmen
      ↓
Objekte erkennen
      ↓
Objektposition berechnen
      ↓
Dobot bewegt sich
      ↓
Objekt aufnehmen
      ↓
Dobot steht wieder still
      ↓
neues Bild aufnehmen

Variante B: permanenter Videostream

Ein Videostream eignet sich für Live-Ansicht, Einrichtung, Testen und Überwachung.

import cv2

stream_url = "http://192.168.1.123:81/stream"

kamera = cv2.VideoCapture(stream_url)

while True:
    erfolgreich, bild = kamera.read()

    if not erfolgreich:
        break

    cv2.imshow("ESP32-CAM", bild)

    if cv2.waitKey(1) == 27:
        break

kamera.release()
cv2.destroyAllWindows()

Eine sinnvolle Aufteilung wäre:

ModusAufgabe
Live-ModusVideostream anzeigen
Erkennungs-ModusGezielt ein Einzelbild aufnehmen
Roboter-ModusEinzelbild → Erkennung → Bewegung → Kontrollbild

3. Bildübertragung nach Python

Für die Verarbeitung eines Einzelbildes kann Python das JPEG-Bild direkt per HTTP abrufen und mit OpenCV dekodieren.

import cv2
import numpy as np
import requests


ESP32_CAM_URL = "http://192.168.1.123/capture"


def bild_aufnehmen():
    antwort = requests.get(
        ESP32_CAM_URL,
        timeout=5
    )

    antwort.raise_for_status()

    daten = np.frombuffer(
        antwort.content,
        dtype=np.uint8
    )

    bild = cv2.imdecode(
        daten,
        cv2.IMREAD_COLOR
    )

    if bild is None:
        raise RuntimeError(
            "Das Kamerabild konnte nicht dekodiert werden."
        )

    return bild

Die Variable bild enthält danach ein normales OpenCV-Bild und kann direkt weiterverarbeitet werden.

4. Rollen von OpenCV und YOLO

OpenCV

OpenCV eignet sich besonders für:

  • Bild laden und zuschneiden,
  • Helligkeit und Kontrast korrigieren,
  • Farben erkennen,
  • Konturen bestimmen,
  • Kreise und Kanten erkennen,
  • Mittelpunkte berechnen,
  • Winkel bestimmen,
  • Kameraverzerrung korrigieren,
  • Pixelkoordinaten in Roboterkoordinaten umrechnen.

YOLO

YOLO übernimmt die gelernte Objekterkennung. Das Modell kann beispielsweise unterscheiden:

Das ist ein roter Würfel.
Das ist ein blauer Zylinder.
Das ist ein Behälter.
Das ist Werkstück A.
Das ist Werkstück B.

Zusätzlich liefert YOLO die Lage des erkannten Objektes im Bild:

Klasse: Zylinder
Sicherheit: 96 %

Bounding Box:
x1 = 320
y1 = 180
x2 = 420
y2 = 300

Der Mittelpunkt ergibt sich daraus zu:

u = (x1 + x2) / 2
v = (y1 + y2) / 2

6. Erkennung eines Objekts

from ultralytics import YOLO


modell = YOLO("best.pt")


def objekte_erkennen(bild, mindest_sicherheit=0.60):

    ergebnisse = modell.predict(
        source=bild,
        conf=mindest_sicherheit,
        verbose=False
    )

    erkannte_objekte = []

    for ergebnis in ergebnisse:

        for box in ergebnis.boxes:

            klasse_id = int(box.cls[0])
            sicherheit = float(box.conf[0])

            x1, y1, x2, y2 = (
                box.xyxy[0]
                .cpu()
                .tolist()
            )

            mitte_x = (x1 + x2) / 2
            mitte_y = (y1 + y2) / 2

            klassenname = ergebnis.names[klasse_id]

            objekt = {
                "klasse": klassenname,
                "sicherheit": sicherheit,
                "pixel_x": mitte_x,
                "pixel_y": mitte_y,
                "box": (x1, y1, x2, y2)
            }

            erkannte_objekte.append(objekt)

    return erkannte_objekte

Beispiel für das Ergebnis:

[
    {
        "klasse": "roter_wuerfel",
        "sicherheit": 0.94,
        "pixel_x": 417.5,
        "pixel_y": 283.0,
        "box": (375, 240, 460, 326)
    },
    {
        "klasse": "blauer_zylinder",
        "sicherheit": 0.88,
        "pixel_x": 632.0,
        "pixel_y": 391.5,
        "box": (590, 344, 674, 439)
    }
]

7. Vom Kamerapixel zur Dobot-Koordinate

Die Kamera liefert Pixelkoordinaten:

Pixel:
u = 417
v = 283

Der Dobot benötigt dagegen Roboterkoordinaten:

x = 215.3 mm
y = -126.7 mm
z = ...
r = ...

Kamerakoordinatensystem

(0,0) ───────────────────────────► u
  │
  │
  │             Objekt
  │               ●
  │           (u = 417,
  │            v = 283)
  │
  ▼
  v

Dobot-Koordinatensystem

                 y
                 ▲
                 │
                 │
                 │
                 └──────────────► x

Die Verbindung zwischen beiden Koordinatensystemen ist der wichtigste geometrische Teil des Projekts.

8. Lochrasterplatte als Zwischenkoordinatensystem

Für das vorhandene Dobot-Projekt ist folgende Dreiteilung besonders sinnvoll:

Kamerabild
Pixel (u, v)
     │
     ▼
Lochrasterplatte
Plattenkoordinate (Spalte, Zeile)
     │
     ▼
Dobot
Roboterkoordinate (x, y)

Damit bleiben Kamera, Platte und Roboter logisch getrennt.

Beispiel:

Pixel:
(417, 283)

entspricht:

Spalte = 12.7
Zeile  = 8.4

Danach übernimmt dobot.py die Umrechnung in die tatsächlichen Roboterkoordinaten.

x, y = dobot.plattenkoordinate(
    spalte=12.73,
    zeile=8.41
)
Für die Bilderkennung sollten die Plattenkoordinaten auch Fließkommazahlen zulassen. Damit können Objekte zwischen den Löchern exakt angefahren werden.

9. Perspektivische Transformation

Auch bei scheinbar senkrechter Kamera ist das Bild meist nicht perfekt maßstabsgleich. Schon eine leichte Schrägstellung verändert den Maßstab innerhalb des Bildes.

oben:    1 Lochabstand = 25 Pixel
unten:   1 Lochabstand = 31 Pixel

Eine einfache Multiplikation mit einem festen Faktor reicht dann nicht aus.

Direkte Transformation von Bildpunkten zu Dobot-Punkten

import cv2
import numpy as np


bildpunkte = np.float32([
    [120, 100],
    [1100, 105],
    [115, 700],
    [1090, 695]
])


dobotpunkte = np.float32([
    [-73.8, -311.1],
    [x2, y2],
    [x3, y3],
    [x4, y4]
])


transformationsmatrix = cv2.getPerspectiveTransform(
    bildpunkte,
    dobotpunkte
)

11. Beispiel für den Ablauf in start.py

import dobot

from bilderkennung.kamera import bild_aufnehmen
from bilderkennung.erkennung import objekte_erkennen
from bilderkennung.kalibrierung import pixel_zu_platte


def hauptprogramm():

    # 1. Dobot verbinden
    dobot.verbinden()

    # 2. Home-Fahrt
    dobot.home()

    # 3. Bild aufnehmen
    bild = bild_aufnehmen()

    # 4. Objekte erkennen
    objekte = objekte_erkennen(bild)

    # 5. Ergebnisse ausgeben
    for objekt in objekte:

        print(
            objekt["klasse"],
            objekt["sicherheit"],
            objekt["pixel_x"],
            objekt["pixel_y"]
        )

    # 6. Gewünschtes Objekt auswählen
    objekt = objekte[0]

    # 7. Pixelposition umrechnen
    spalte, zeile = pixel_zu_platte(
        objekt["pixel_x"],
        objekt["pixel_y"]
    )

    # 8. Dobot-Koordinaten bestimmen
    x, y = dobot.plattenkoordinate(
        spalte,
        zeile
    )

    # 9. Objekt anfahren
    dobot.fahre_zu(
        x=x,
        y=y,
        z=20
    )
Dieses Beispiel zeigt die Architektur. Für den realen Roboterbetrieb müssen Sicherheitsbewegungen, Reichweitenprüfung, Fehlerbehandlung und Greiflogik ergänzt werden.

12. Sicherer Roboterablauf

Eine erkannte YOLO-Position sollte niemals ungeprüft direkt in eine Abwärtsbewegung umgesetzt werden.

  1. Objekt erkennen.
  2. Plausibilität prüfen: bekannte Klasse, ausreichende Sicherheit, erlaubter Arbeitsbereich, erreichbare Koordinate.
  3. Sicherheitsposition oberhalb des Objektes anfahren.
  4. Position bei Bedarf erneut prüfen.
  5. Absenken.
  6. Sauger oder Greifer einschalten.
  7. Anheben.
  8. Zielposition anfahren.
  9. Absenken.
  10. Sauger oder Greifer ausschalten.
  11. Anheben.
  12. Kontrollbild aufnehmen.

Beispiel:

Erkannte Position:

x = 210 mm
y = -120 mm

Greifhöhe:
z = -35 mm

Sicherheitshöhe:
z = +30 mm


Bewegung:

(210, -120, +30)
        ↓
(210, -120, -35)
        ↓
Sauger EIN
        ↓
(210, -120, +30)
Die Bilderkennung ist ein Sensor. Die Robotersteuerung entscheidet, ob und wie eine erkannte Position tatsächlich angefahren werden darf.

13. Welche Objekte können erkannt werden?

Einfache ebene Formen

Für Kreis, Dreieck, Quadrat oder Rechteck reicht häufig klassische Bildverarbeitung mit OpenCV.

Graustufen
      ↓
Schwellwert
      ↓
Kontur
      ↓
Anzahl der Ecken
      ↓
Form bestimmen

Räumliche Körper

Für Klassen wie Würfel, Quader, Zylinder oder Kegel kann YOLO mit trainierten Beispielen eingesetzt werden.

Klasse 0 = Würfel
Klasse 1 = Quader
Klasse 2 = Zylinder
Klasse 3 = Kegel

Dazu werden Trainingsbilder aus unterschiedlichen Positionen, Drehwinkeln, Beleuchtungen, Abständen und Hintergründen benötigt.

14. Hybride Erkennung: YOLO + OpenCV

Die Kombination aus YOLO und OpenCV ist besonders leistungsfähig:

YOLO
  ↓
Welche Art von Objekt ist das?

OpenCV
  ↓
Wo liegt der geometrisch beste Greifpunkt?
Wie ist das Objekt gedreht?

Dobot
  ↓
Objekt aufnehmen

Beispiel:

YOLO:

„Das ist ein Quader.“

Bounding Box:
x1 = 300
y1 = 200
x2 = 500
y2 = 350

OpenCV kann innerhalb dieses Bildausschnittes den Mittelpunkt und gegebenenfalls den Drehwinkel genauer bestimmen.

Mittelpunkt:
(u, v)

Drehwinkel:
37°

Danach kann der Dobot beispielsweise folgende Zielwerte erhalten:

x = 213.2
y = -118.7
r = 37.0

Für reine Sauggreif-Aufgaben kann r zunächst konstant bleiben.

15. Das Problem der Z-Koordinate

Eine einzelne ESP32-CAM liefert zunächst nur zweidimensionale Bildkoordinaten:

u
v

Der Dobot benötigt jedoch:

x
y
z
r

Lösung A: bekannte Objekthöhe

OBJEKTHOEHEN = {
    "wuerfel": 30.0,
    "zylinder": 50.0,
    "quader": 40.0
}

Lösung B: alle Objekte gleich hoch

GREIF_Z = -35.0

Lösung C: echte 3D-Erfassung

Dafür wären zusätzliche Sensoren nötig, beispielsweise:

  • zweite Kamera,
  • Stereo-Kamera,
  • Tiefenkamera,
  • Time-of-Flight-Sensor.
Für den ersten Aufbau ist eine bekannte oder konstante Greifhöhe die sinnvollste Lösung.

16. Die Lochrasterplatte als Kalibriermuster

Die vorhandene Platte mit 40 Spalten × 27 Zeilen bietet bereits ein sehr gutes geometrisches Bezugssystem.

Beispielsweise können vier bekannte Löcher verwendet werden:

Loch (2, 2)
Loch (39, 2)
Loch (2, 26)
Loch (39, 26)

Deren Pixelpositionen werden im Kamerabild bestimmt:

BILDPUNKTE = [
    [124, 88],
    [1120, 93],
    [130, 704],
    [1111, 698]
]

Dazu sind die Plattenkoordinaten bekannt:

PLATTENPUNKTE = [
    [2, 2],
    [39, 2],
    [2, 26],
    [39, 26]
]

Damit entsteht die Zuordnung:

Pixel
  ↓
Lochrasterkoordinate

Langfristig könnten zusätzliche Markierungen oder ArUco-Marker an der Platte angebracht werden, um die Lage der Kamera automatisch zu kontrollieren.

18. Empfohlene Umsetzung in Stufen

  1. ESP32-CAM liefert ein Bild an Python.
  2. OpenCV zeigt das Bild an.
  3. Ein Mausklick im Bild liefert die Pixelposition.
  4. Die Pixelposition wird in eine Position auf der Lochrasterplatte umgerechnet.
  5. Ein Mausklick im Kamerabild erzeugt eine Dobot-Zielposition.
  6. Einfache OpenCV-Objekterkennung ergänzen.
  7. YOLO erkennt Objekte automatisch.
  8. YOLO + OpenCV + Dobot führen automatisches Pick-and-Place aus.
  9. Integration in containerlager.py.
Der wichtigste Zwischenschritt ist: Auf ein Objekt im Kamerabild klicken und den Dobot genau zu dieser Position fahren lassen.

Wenn diese geometrische Verbindung zuverlässig funktioniert, kann die Zielposition später beliebig erzeugt werden, zum Beispiel durch:

Mausklick
OpenCV
YOLO
QR-Code
ArUco-Marker
Farberkennung
