Dobot Magician – Eigene Python-Hilfsfunktionen

Dokumentation der in mein_programm.py definierten Komfort- und Hilfsfunktionen

Inhaltsverzeichnis

Grundlage 1. warten_bis_fertig() 2. queue_starten() 3. queue_stoppen() 4. position_lesen() 5. position_anzeigen() 6. fahre_zu() 7. home() 8. test_z() 9. ausfuehren() 10. main() Zusammenhang der Funktionen Komplettes Beispiel

Grundlage

Die hier beschriebenen Funktionen gehören nicht direkt zur offiziellen Dobot-API. Sie wurden in mein_programm.py als eigene Python-Hilfsfunktionen definiert.

Sie vereinfachen häufig benötigte Abläufe. Statt mehrere API-Aufrufe jedes Mal neu zu schreiben, kann später beispielsweise einfach verwendet werden:

fahre_zu(api, dType, 200, 0, 80, 0)

Die Funktionen bilden also eine zusätzliche Ebene über DobotDllType.py:

Eigenes Roboterprogramm ↓ Eigene Hilfsfunktionen in mein_programm.py ↓ DobotDllType.py ↓ DobotDll.dll ↓ Dobot Magician
Wichtig: Diese Funktionen können jederzeit erweitert oder verändert werden. Sie sind unsere eigene Programmierschnittstelle für den Dobot.

1. warten_bis_fertig()

Aufgabe

Wartet, bis ein zuvor in die Dobot-Queue eingetragener Befehl vollständig ausgeführt wurde.

Definition

def warten_bis_fertig(api, dType, ziel_index):
    """Wartet, bis ein Queue-Befehl vollständig ausgeführt wurde."""

    while dType.GetQueuedCmdCurrentIndex(api)[0] < ziel_index:
        dType.dSleep(100)

Parameter

ParameterBedeutung
apiAPI-Objekt des verbundenen Dobot
dTypeDas importierte Modul DobotDllType
ziel_indexQueue-Index des Befehls, auf dessen Abschluss gewartet wird

Beispiel

ziel_index = dType.SetPTPCmd(
    api,
    dType.PTPMode.PTPMOVJXYZMode,
    200,
    0,
    80,
    0,
    isQueued=1
)[0]

warten_bis_fertig(api, dType, ziel_index)

Wirkungsweise

Die Funktion fragt alle 100 ms den aktuellen Queue-Index des Dobot ab. Erst wenn der Zielindex erreicht wurde, wird die Funktion beendet.

2. queue_starten()

Aufgabe

Bereitet die Dobot-Befehlswarteschlange für einen neuen Programmablauf vor.

Definition

def queue_starten(api, dType):
    """Stoppt eine laufende Queue, löscht sie und startet sie neu."""

    dType.SetQueuedCmdStopExec(api)
    dType.SetQueuedCmdClear(api)
    dType.SetQueuedCmdStartExec(api)

Ablauf

  1. Eine eventuell laufende Queue wird gestoppt.
  2. Vorhandene alte Befehle werden gelöscht.
  3. Die Queue wird neu gestartet.

Aufruf

queue_starten(api, dType)
Diese Funktion sorgt dafür, dass ein neues Roboterprogramm nicht versehentlich alte Befehle aus einer früheren Queue weiter ausführt.

3. queue_stoppen()

Aufgabe

Stoppt die Ausführung der Dobot-Befehlswarteschlange.

Definition

def queue_stoppen(api, dType):
    """Stoppt die Ausführung der Queue."""

    dType.SetQueuedCmdStopExec(api)

Aufruf

queue_stoppen(api, dType)

Die Funktion wird insbesondere am Ende von ausfuehren() verwendet.

4. position_lesen()

Aufgabe

Liest die aktuelle kartesische Position des Dobot.

Definition

def position_lesen(api, dType):
    """Liest die kartesische Position X, Y, Z und R."""

    return dType.GetPose(api)[:4]

Rückgabewert

Die Funktion liefert vier Werte zurück:

X, Y, Z, R

Beispiel

x, y, z, r = position_lesen(api, dType)

print(x)
print(y)
print(z)
print(r)

Warum diese Hilfsfunktion?

dType.GetPose(api) liefert insgesamt acht Werte. Für viele kartesische Bewegungen werden jedoch nur die ersten vier Werte benötigt.

dType.GetPose(api)
# liefert:
# X, Y, Z, R, J1, J2, J3, J4

position_lesen() beschränkt die Rückgabe deshalb auf:

dType.GetPose(api)[:4]

5. position_anzeigen()

Aufgabe

Liest die aktuelle Position und gibt sie formatiert auf dem Bildschirm aus.

Definition

def position_anzeigen(api, dType):
    """Liest und zeigt die aktuelle kartesische Position an."""

    x, y, z, r = position_lesen(api, dType)

    print(
        f"X={x:.1f} mm, "
        f"Y={y:.1f} mm, "
        f"Z={z:.1f} mm, "
        f"R={r:.1f}°"
    )

Aufruf

position_anzeigen(api, dType)

Beispielausgabe

X=200.0 mm, Y=0.0 mm, Z=80.0 mm, R=0.0°

Die Funktion verwendet intern position_lesen().

6. fahre_zu()

Aufgabe

Fährt den Dobot zu einer Zielposition und wartet automatisch, bis die Bewegung abgeschlossen ist.

Definition

def fahre_zu(api, dType, x, y, z, r, modus=None):
    """Fährt zu einer Zielposition und wartet auf das Ende der Bewegung."""

    if modus is None:
        modus = dType.PTPMode.PTPMOVJXYZMode

    ziel_index = dType.SetPTPCmd(
        api,
        modus,
        x,
        y,
        z,
        r,
        isQueued=1
    )[0]

    warten_bis_fertig(api, dType, ziel_index)

Parameter

ParameterBedeutung
apiAPI-Objekt des Dobot
dTypeModul DobotDllType
xZielposition X
yZielposition Y
zZielposition Z
rZielrotation R
modusPTP-Bewegungsmodus; optional

Standardmodus

Wird kein Modus angegeben, verwendet die Funktion:

dType.PTPMode.PTPMOVJXYZMode

Einfacher Aufruf

fahre_zu(
    api,
    dType,
    200,
    0,
    80,
    0
)

Lineare Bewegung

fahre_zu(
    api,
    dType,
    200,
    0,
    50,
    0,
    dType.PTPMode.PTPMOVLXYZMode
)

Nur die Z-Koordinate verändern

x, y, z, r = position_lesen(api, dType)

fahre_zu(
    api,
    dType,
    x,
    y,
    z - 20,
    r,
    dType.PTPMode.PTPMOVLXYZMode
)
Achtung: fahre_zu() führt eine reale Roboterbewegung aus. Zielkoordinaten und Arbeitsbereich müssen vorher geprüft werden.

Vorteil gegenüber direktem API-Aufruf

Ohne die Hilfsfunktion müsste immer Folgendes geschrieben werden:

ziel_index = dType.SetPTPCmd(
    api,
    dType.PTPMode.PTPMOVJXYZMode,
    200,
    0,
    80,
    0,
    isQueued=1
)[0]

warten_bis_fertig(api, dType, ziel_index)

Mit der Hilfsfunktion genügt:

fahre_zu(api, dType, 200, 0, 80, 0)

7. home()

Aufgabe

Startet die HOME-Fahrt des Dobot und wartet automatisch auf deren Abschluss.

Definition

def home(api, dType):
    """Führt eine HOME-Fahrt aus und wartet auf deren Abschluss."""

    print("HOME-Fahrt wird gestartet.")

    ziel_index = dType.SetHOMECmd(
        api,
        0,
        1
    )[0]

    warten_bis_fertig(api, dType, ziel_index)

    print("HOME-Fahrt abgeschlossen.")

Aufruf

home(api, dType)
Achtung: Die Funktion verwendet die aktuell im Dobot eingestellte HOME-Position. Vor dem Aufruf muss geprüft werden, ob diese Position mit dem montierten Werkzeug sicher erreichbar ist.

8. test_z()

Aufgabe

Erlaubt das interaktive Testen verschiedener Z-Koordinaten. X, Y und R bleiben dabei unverändert.

Aufruf

test_z(api, dType)

Ablauf

  1. Aktuelle Position wird gelesen.
  2. Der Benutzer gibt eine neue Z-Koordinate ein.
  3. Der Dobot fährt linear auf diese neue Z-Höhe.
  4. Die tatsächlich erreichte Position wird erneut gelesen.
  5. Mit a wird der Test beendet.

Kern des Bewegungsaufrufs

fahre_zu(
    api,
    dType,
    x,
    y,
    neue_z,
    r,
    dType.PTPMode.PTPMOVLXYZMode
)

Beispiel für die Eingabe

Position: X=200.0, Y=0.0, Z=80.0, R=0.0
Neue Z-Koordinate oder 'a' zum Abbrechen: 60
Achtung: Bei zu kleinen Z-Werten kann das Werkzeug auf die Arbeitsplatte oder andere Gegenstände treffen.

9. ausfuehren()

Aufgabe

Ist die zentrale Startfunktion des eigentlichen Roboterprogramms.

Sie wird von start.py aufgerufen:

import mein_programm

mein_programm.ausfuehren(api, dType)

Definition

def ausfuehren(api, dType):
    """Wird von start.py aufgerufen."""

    print()
    print("Roboterprogramm wird gestartet.")
    print("--------------------------------")

    queue_starten(api, dType)

    try:
        print("Aktuelle Position:")
        position_anzeigen(api, dType)

        test_z(api, dType)

    finally:
        queue_stoppen(api, dType)

        print()
        print("Roboterprogramm beendet.")

Ablauf

  1. Queue vorbereiten und starten
  2. Aktuelle Position anzeigen
  3. Eigentlichen Programmablauf ausführen
  4. Queue im finally-Block sicher stoppen
Der eigentliche spätere Roboterablauf wird innerhalb des try-Blocks ergänzt.

Beispiel für einen späteren Ablauf

def ausfuehren(api, dType):
    queue_starten(api, dType)

    try:
        home(api, dType)

        fahre_zu(
            api,
            dType,
            200,
            0,
            80,
            0
        )

    finally:
        queue_stoppen(api, dType)

10. main()

Aufgabe

Verhindert nicht den direkten Start der Datei, weist aber darauf hin, dass normalerweise start.py verwendet werden soll.

Definition

def main():
    print(
        "Diese Datei wird normalerweise nicht direkt gestartet.\n"
        "Bitte 'start.py' ausführen."
    )


if __name__ == "__main__":
    main()

Normaler Programmstart

start.py
    ↓
mein_programm.py
    ↓
ausfuehren(api, dType)

Zusammenhang der Funktionen

ausfuehren() │ ├── queue_starten() │ ├── position_anzeigen() │ └── position_lesen() │ ├── test_z() │ ├── position_lesen() │ └── fahre_zu() │ └── warten_bis_fertig() │ ├── home() │ └── warten_bis_fertig() │ └── queue_stoppen()

Die Hilfsfunktionen bauen also teilweise aufeinander auf. Besonders wichtig ist:

fahre_zu()
    ↓
SetPTPCmd()
    ↓
warten_bis_fertig()

Komplettes Beispiel

Ein späteres Roboterprogramm kann mit den eigenen Hilfsfunktionen sehr kompakt aussehen:

def ausfuehren(api, dType):
    queue_starten(api, dType)

    try:
        print("Startposition:")
        position_anzeigen(api, dType)

        # HOME-Fahrt
        home(api, dType)

        # Aktuelle Position lesen
        x, y, z, r = position_lesen(api, dType)

        # 20 mm nach oben fahren
        fahre_zu(
            api,
            dType,
            x,
            y,
            z + 20,
            r,
            dType.PTPMode.PTPMOVLXYZMode
        )

        print("Endposition:")
        position_anzeigen(api, dType)

    finally:
        queue_stoppen(api, dType)

Gedanke hinter diesen Funktionen

Die Funktionen bilden eine kleine, eigene Python-Bibliothek für den Dobot. Dadurch werden die eigentlichen Roboterprogramme kürzer, übersichtlicher und für Schüler leichter lesbar.

Beispielsweise ist:

fahre_zu(api, dType, 200, 0, 80, 0)

deutlich einfacher zu verstehen als die vollständige Kombination aus SetPTPCmd(), Queue-Index und Warteschleife.
