Shelly Plus Add-on: Werte per HTTP-Request abfragen

Kurzübersicht für Shelly Plus 1 / Shelly Gen2-RPC-Kommandos

Grundidee

Das Shelly Plus Add-on erscheint nicht als ein einzelnes Messgerät mit einem eigenen Statusbefehl. Die angeschlossenen Sensoren und Eingänge werden als normale Shelly-RPC-Komponenten angezeigt, zum Beispiel als temperature:100, humidity:100, input:100 oder voltmeter:100.

Wichtig: Die ID ist nicht immer 100. Am zuverlässigsten ist zuerst eine Gesamtabfrage mit Shelly.GetStatus oder eine Peripherie-Abfrage mit SensorAddon.GetPeripherals.

Wichtigster Sammel-Befehl

http://<IP-DES-SHELLY>/rpc/Shelly.GetStatus

Beispiel im Access-Point-Modus:

http://192.168.33.1/rpc/Shelly.GetStatus

Dieser Befehl liefert den Status aller vorhandenen Komponenten, zum Beispiel:

{
  "switch:0": { ... },
  "input:0": { ... },
  "temperature:100": { ... },
  "humidity:100": { ... },
  "voltmeter:100": { ... }
}

Einzelbeispiele

Temperatur abfragen

http://<IP>/rpc/Temperature.GetStatus?id=100
{
  "id": 100,
  "tC": 22.5,
  "tF": 72.5
}

Luftfeuchtigkeit abfragen

http://<IP>/rpc/Humidity.GetStatus?id=100
{
  "id": 100,
  "rh": 48.3
}

Digitalen Eingang abfragen

http://<IP>/rpc/Input.GetStatus?id=100
{
  "id": 100,
  "state": true
}

http://<IP>/rpc/Voltmeter.GetStatus?id=100
{
  "id": 100,
  "voltage": 3.27
}

Add-on-Verwaltung

Welche Sensoren und Eingänge am Add-on eingerichtet sind, lässt sich mit folgendem RPC-Befehl prüfen:

http://<IP>/rpc/SensorAddon.GetPeripherals

Für 1-Wire-Sensoren, zum Beispiel DS18B20, ist außerdem ein Scan möglich:

http://<IP>/rpc/SensorAddon.OneWireScan

Praktische Test-Reihenfolge

  1. Gesamtstatus öffnen: http://<IP>/rpc/Shelly.GetStatus
  2. Im Ergebnis nach temperature:, humidity:, input: und voltmeter: suchen.
  3. Die gefundene ID übernehmen, zum Beispiel temperature:100 → id=100.
  4. Danach gezielt mit Temperature.GetStatus, Humidity.GetStatus, Input.GetStatus oder Voltmeter.GetStatus abfragen.

Kurzfazit

Für den Unterricht und die Fehlersuche ist Shelly.GetStatus der beste Einstieg. Danach werden die gefundenen Komponenten einzeln abgefragt. Die häufigste Stolperfalle ist die ID: Nicht blind id=100 verwenden, sondern zuerst im Status nachsehen.
