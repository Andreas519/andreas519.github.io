    andreas519.github.io - Actions

typischer Ablauf

  1. Änderung lokal speichern
  2. Commit in GitHub Desktop
  3. Push zu GitHub
  4. GitHub Pages baut/veröffentlicht die Seite neu - Browser zeigt eventuell noch alte Version aus dem Cache
  5. Im privaten Fenster testen - Damit umgehst du oft den Browser-Cache.
Actions prüfen - Im Repository: GitHub → Repository → Actions
Dort siehst du, ob „pages build and deployment“ noch läuft oder erfolgreich abgeschlossen ist.
