ESP32 Devboard Varianten

Die hier gezeigten Boards verwenden meist ein ESP32-WROOM-32 Modul. Das ist ein weit verbreitetes ESP32-Modul mit WLAN/Bluetooth, das auf vielen günstigen Entwicklungsboards eingesetzt wird.

Typische Komponenten dieser Boards:

  • ESP32-WROOM-32 Modul mit Dual-Core-MCU.
  • USB-Serial-Bridge (z. B. CH340, CP2102, FT232), damit das Board über USB programmiert werden kann.
  • Spannungsregler für 5V/USB auf 3,3V.
  • Reset-Taste (EN) und oft eine Boot-Taste für den Flash-Modus.
  • Stiftleisten mit GPIO-Zugriff und Signalbeschriftungen.

Der wichtigste Unterschied liegt in der Anzahl der Stiftleisten-Pins:

  • 30-Pin-Board: kompakter, niedrigere Gesamtbreite, oft einfacher in Steckbretter einzupassen.
  • 38-Pin-Board: mehr GPIO-Pins nach außen geführt, besser geeignet für mehrere Sensoren, Displays oder Shields.

Beide Varianten haben üblicherweise folgende Anschlüsse:

  • 3V3 für die 3,3‑V-Versorgung
  • GND als Masseanschluss
  • EN für den Reset des ESP32
  • TX0(GPIO1) und RX0(GPIO2) für die serielle USB-Kommunikation
  • SCL(GPIO22) und SCL(GPIO21) für die I2C-Kommunikation
  • ADC-fähige Pins für analoge Sensoren

Tipps für den Einsatz

  1. Verbinde das Board mit USB-C oder Micro-USB (je nach Modell), um die 5‑V‑Versorgung bereitzustellen.
  2. Achte auf die Beschriftung der Pins, insbesondere wenn das Board nur einseitig gekennzeichnet ist.
  3. Nutze die Pinout-Bilder als Referenz, bevor du Kabel anschließt.
  4. Verschaffe die für den von dir verwendeten ESP ein entsprechendes Pinout.
    Achte auf die Anordnung der Teile auf dem Board und auf die Beschiftung.
  5. Dokumentiere die Bezeichnungen der verwendeten Pins ausführlich im Programmquelltext. Beispiele: 
    // Arduino (C++) - Beispiel
  // i2c-Modul SDA  --- gelb --- D21 - GPIO21 - SDA
  // i2c-Modul SCL  --- grün --- D22 - GPIO22 - SCL
  const int I2C_SDA = 21; // gelb, D21
  const int I2C_SCL = 22; // grün, D22

  # MicroPython - Beispiel
  # i2c-Modul SDA  --- gelb --- D21 - GPIO21 - SDA
  # i2c-Modul SCL  --- grün --- D22 - GPIO22 - SCL
  # from machine import Pin, I2C
  # i2c = I2C(0, scl=Pin(22), sda=Pin(21))

Mit der richtigen Pinbelegung lassen sich WLAN, Bluetooth, Sensoren und Aktoren schnell in Projekte integrieren.
